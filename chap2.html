Folder Structure

In this chapter, let's look into the files and folders that are created and how the command flows.

From the last chapter, we know that when the command 'npm create-react-app' is run, an entire project is created and would look something like this:
As you can see, there are 3 folders namely node_modules, public and src and 4 files namely gitignore, package-lock.json, package.json and README.md
Among these, let's discuss deeply on 4 important files.
package.json
index.html
index.js
App.js
1. package.json

This file containes the dependencies and the script needed for the project.

Here as you can see, we are using react version 16.13, which is under dependencies. We can add more dependencies by installing the necessary ones using npm, which we'll be seeing in the upcoming chapters.
As of scripts, we have scripts to run, build and run tests.

2. index.html

This is going to be the only the HTML you'll use in the whole react application. The idea is to build a single page HTML, and the view is going to change dynamically in the browser.

We are not going to add any code to this file. We want React to control how the application is going to look like. To make that possible, we are going to have one div tag, with a specific id. Now React takes over this id and is responsible for the UI of the application.

3. index.js

This is going to be the starting point of our React application. Here the root component is specified which is 'App' component and the DOM element, which is also the id name in 'index.html'.

Here the id name is 'root'. This element will be completely controlled by React through the component 'App'.

4. App.js

React just like any javascript file works using a function, and this function, in turn, decides the UI of the application.

Along with this 'App.js' also has an 'App.css' which helps with the styling of the application.

Let's now see how the command-line flows.
When you run the command 'npm start', index.html is served in the browser. index.html contains the root DOM node. From there, the control enters index.js where ReactDOM renders the App component onto the root element. The App component contains the HTML, which is ultimately displayed in the browser.
By now, we have mentioned component several times already, in the next chapter, let's talk about components in detail.
